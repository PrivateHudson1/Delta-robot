
# Протокол связи

## Введение

При разработке данного дельта-робота возникла необходимость в протоколе обмена между управляющим ПК и контроллером OpenCM 904. Готовые промышленные протоколы (Modbus RTU, CANopen, Profinet) либо слишком тяжеловесны для микроконтроллера с 20 КБ RAM, либо требуют внешних шлюзов, либо не обеспечивают нужной гибкости при отладке.

Был разработан собственный двухрежимный протокол, который сочетает:
- **Текстовый режим** — для настройки, отладки и ручного управления. Команды читаются человеком, что позволяет тестировать робота через любой Serial-терминал.
- **Бинарный режим** — для промышленной эксплуатации. Компактный формат с CRC-контролем, адресацией устройств и подтверждением доставки.

Протокол построен по принципу «запрос — ответ». ПК отправляет команду, робот выполняет её и возвращает статус. В бинарном режиме используется CRC16 (Modbus) для обнаружения ошибок в канале. Структура пакета спроектирована так, чтобы в будущем можно было добавить совместимость с Modbus RTU через простой программный шлюз на ПК.

**Общий принцип работы:**

ПК формирует пакет (текстовую строку или бинарную последовательность), отправляет через USB-порт. OpenCM принимает, парсит, выполняет команду (движение, включение вакуума, запрос позиции) и отправляет ответ. Цикл управления роботом работает асинхронно на частоте 200 Гц, а обработка команд происходит в фоне, не блокируя движение.

## Режимы

| Режим | Назначение | Особенности |
|-------|------------|-------------|
| Текстовый | отладка, ручное управление | человекочитаемый, без CRC |
| Бинарный | промышленная эксплуатация | CRC16, адресация, компактный |

Переключение: `PROTOBIN` / `PROTOTEXT`

## Текстовый протокол

**Формат:** `[КОМАНДА] [ПАРАМЕТРЫ]\n`

### Команды

| Команда | Параметры | Пример |
|---------|-----------|--------|
| `MOV` | X Y Z (мм) | `MOV 150 -200 -50` |
| `MOVR` | DX DY DZ (мм) | `MOVR 10 0 0` |
| `STOP` | - | `STOP` |
| `POS?` | - | `POS?` |
| `VAC ON` | - | `VAC ON` |
| `VAC OFF` | - | `VAC OFF` |
| `HOME` | - | `HOME` |
| `STATUS?` | - | `STATUS?` |
| `RESET` | - | `RESET` |
| `SPEED` | V (мм/с) A (мм/с²) | `SPEED 500 2000` |
| `SETPID` | ID (1-3) P I D | `SETPID 1 800 0 4000` |
| `PROTOBIN` | - | `PROTOBIN` |
| `PROTOTEXT` | - | `PROTOTEXT` |

### Ответы

| Ответ | Формат | Пример |
|-------|--------|--------|
| позиция | `POS:x,y,z` | `POS:150.00,-200.00,-50.00` |
| статус | `STATUS:state,estop,error` | `STATUS:BUSY,OK,None` |
| подтверждение | `ACK:cmd,OK/FAIL` | `ACK:1,OK` |
| ошибка | `ERROR:message` | `ERROR:Target unreachable` |

**state:** READY / BUSY
**estop:** OK / ESTOP
**cmd:** числовой код команды

## Бинарный протокол

### Формат пакета

┌───────┬───────┬───────┬───────┬──────────┬─────────┬───────┐
│ 0xAA │ LEN │ ADDR │ CMD │ DATA │ CRC16 │ 0x55 │
│ 1 байт│ 1 байт│ 1 байт│ 1 байт│ 0-32 байт│ 2 байта │ 1 байт│
└───────┴───────┴───────┴───────┴──────────┴─────────┴───────┘


### Поля

| Поле | Описание |
|------|----------|
| `0xAA` | стартовый байт |
| `LEN` | длина данных + 4 (addr+cmd+reserved+data) |
| `ADDR` | адрес устройства (1-255, 255 = broadcast) |
| `CMD` | код команды |
| `DATA` | данные (0-32 байта) |
| `CRC16` | Modbus CRC-16, little-endian |
| `0x55` | конечный байт |

### Коды команд

| Код | Команда | Данные | Размер |
|-----|---------|--------|--------|
| 0x10 | MOVE_ABS | X, Y, Z (float) | 12 |
| 0x11 | MOVE_REL | dX, dY, dZ (float) | 12 |
| 0x12 | STOP | - | 0 |
| 0x13 | GET_POS | - | 0 |
| 0x14 | HOME | - | 0 |
| 0x20 | VAC_ON | - | 0 |
| 0x21 | VAC_OFF | - | 0 |
| 0x30 | SET_SPEED | V (float), A (float) | 8 |

### Коды ответа

| Код | Значение |
|-----|----------|
| 0x00 | OK |
| 0x01 | ERROR |
| 0x02 | INVALID_CMD |
| 0x03 | INVALID_CRC |
| 0x04 | BUSY |
| 0x05 | TIMEOUT |
| 0x06 | UNREACHABLE |

### Структура ответа

[0xAA][LEN][ADDR][CMD][RESP_CODE][CRC16][0x55]

- `RESP_CODE` — код ответа (из таблицы выше)
- `CMD` — эхо команды
- `LEN` = 5 (адрес + команда + код ответа + CRC)

**Запрос движения в (150, -200, -50):**

**Ответ OK:**


**Запрос позиции:**
AA 04 01 13 00 8A 65 55

**Ответ с позицией (150, -200, -50):**
AA 10 01 13 00 00 00 96 00 00 38 00 00 50 41 55


### CRC16 вычисление

Полином: 0xA001 (Modbus)

```python
def crc16(data: bytes) -> int:
    crc = 0xFFFF
    for byte in data:
        crc ^= byte
        for _ in range(8):
            if crc & 1:
                crc = (crc >> 1) ^ 0xA001
            else:
                crc >>= 1
    return crc
```

## Интеграция с Modbus
OpenCM изначально не поддерживает Modbus напрямую. Для этого используется python шлюз-конвертер:

```python
from pymodbus.server import StartTcpServer
class ModbusGateway:
    def __init__(self, robot):
        self.robot = robot
        # маппинг регистров
        self.registers = {
            40001: lambda: self.robot.current_position[0],  # X
            40002: lambda: self.robot.current_position[1],  # Y
            40003: lambda: self.robot.current_position[2],  # Z
            40010: self._exec_move,  # X целевая
        }
    
    def _exec_move(self, value):
        self.robot.move_to(value, ...)
```
